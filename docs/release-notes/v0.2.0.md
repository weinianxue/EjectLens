# EjectLens v0.2.0

## New features

- **Settings dialog** with General, Appearance, and Behavior tabs.
  - Language: English, Simplified Chinese, or system default.
  - Theme: Light, Dark, or system.
  - Window size and position memory.
  - Behavior toggles: confirm before eject, refresh after eject,
    remember last drive, default time range.
  - Settings saved to %APPDATA%\EjectLens\settings.json.
- **Safe Eject button** that requests Windows to safely remove
  the selected drive through the Configuration Manager API.
  - Shows veto type and name when Windows rejects the request.
  - Eject result is included in the diagnostic report.

## Improvements

- Release packaging workflow (`.github/workflows/release.yml`)
  triggered on tag push or manual dispatch.
  Produces `EjectLens-v*-win-x64-portable.zip` with the single-file
  executable and documentation.
- README now explains that Code → Download ZIP is source only.
  Users should download release packages from GitHub Releases.
- MainForm split into partial classes for maintainability.
- CI workflow runs on every push and pull request.

## Fixes

- Corrected MANUAL_STEPS.md to not suggest the release zip is in
  the repository root.
- .gitignore now covers release artifacts (`*.zip`, `*.nupkg`,
  `publish/`).

## Requirements

- Windows 10 or Windows 11
- .NET 8 Desktop Runtime (for framework-dependent builds)
- Self-contained single-file builds are available in the release package.
