# EjectLens v0.2.0

## New features

- **Settings dialog** with General, Appearance, and Behavior tabs.
  - Language: English, Simplified Chinese, or system default.
  - Theme: Light, Dark, or system.
  - Window size and position memory.
  - Behavior toggles: confirm before eject, refresh after eject,
    remember last drive, default time range.
  - Settings saved to %APPDATA%\EjectLens\settings.json.
- **Safe Eject button** that requests Windows to safely remove
  the selected drive through the Configuration Manager API.
  - Walks the device tree upward from the disk node to find
    the correct removable USB parent device.
  - Tries multiple candidate devnodes in priority order.
  - Falls back to safe volume lock-and-eject when all CM_Request_Device_Eject
    calls are vetoed (lock → eject media; never force-dismounts).
  - Shows detailed diagnostics: return code name, veto type/name,
    all attempted devnodes, and suggested next steps.
  - Eject result is included in the diagnostic report.

## Improvements

- Release packaging workflow triggered on tag push or manual dispatch.
  Produces `EjectLens-v0.2.0-win-x64-portable.zip`.
- README explains that Code → Download ZIP is source only.
- MainForm split into partial classes.
- CI workflow runs on every push and pull request.

## Known limitations

- Event ID 225 must exist in the System event log for event-based diagnostics.
- Safe eject depends on Windows and device driver behavior.
- Some USB drives may still return Driver veto on all candidate nodes.
- Administrator privileges may be required for some operations.

## Requirements

- Windows 10 or Windows 11
- .NET 8 Desktop Runtime
